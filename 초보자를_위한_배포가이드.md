# 🎯 초보자를 위한 GitHub Pages 배포 가이드

이 가이드는 프로그래밍을 처음 시작하는 분들을 위해 매우 자세하게 작성되었습니다.
걱정하지 마세요! 차근차근 따라하시면 됩니다. 😊

---

## 📋 준비물

1. **GitHub 계정** (없으면 [github.com](https://github.com)에서 무료로 만들기)
2. **이미 만든 QR 코드 생성기 프로젝트**
3. **인터넷 연결**

---

## 1단계: Git 설치 확인 (5분)

### Git이 설치되어 있는지 확인하기

1. **Windows 키**를 누르고 `cmd` 또는 `명령 프롬프트`를 검색해서 실행
2. 검은 창이 열리면 다음을 입력하고 **Enter** 누르기:
   ```
   git --version
   ```
3. 버전 번호가 나오면 ✅ **설치됨!** (2단계로 넘어가기)
4. "명령을 찾을 수 없습니다" 같은 메시지가 나오면 ❌ **설치 필요**

### Git 설치하기 (필요한 경우만)

1. [git-scm.com/download/win](https://git-scm.com/download/win) 접속
2. 다운로드 버튼 클릭
3. 다운로드된 파일 실행
4. **Next** 버튼만 계속 눌러서 설치 (기본 설정 그대로 사용)
5. 설치 완료 후 컴퓨터 재시작
6. 다시 위의 "Git이 설치되어 있는지 확인하기" 단계 실행

---

## 2단계: 프로젝트 폴더에서 Git 초기화 (2분)

### 명령 프롬프트에서 프로젝트 폴더로 이동하기

1. **Windows 키** + **R** 누르기
2. `cmd` 입력하고 **Enter**
3. 검은 창이 열리면, 프로젝트 폴더 경로로 이동:

   예를 들어 프로젝트가 `D:\9999.Study\cursor\20260115\QR`에 있다면:
   ```
   cd D:\9999.Study\cursor\20260115\QR
   ```
   
   **팁**: 프로젝트 폴더 경로를 모르면:
   - 파일 탐색기에서 프로젝트 폴더 열기
   - 주소창을 클릭하면 경로가 선택됨
   - **Ctrl + C**로 복사
   - 명령 프롬프트에서 `cd ` 입력 후 **Ctrl + V**로 붙여넣기
   - **Enter** 누르기

### Git 초기화하기

프로젝트 폴더로 이동한 후, 다음 명령어들을 **하나씩** 입력하고 **Enter** 누르기:

```bash
git init
```

```bash
git add .
```

```bash
git commit -m "첫 번째 커밋"
```

**각 명령어 입력 후 에러가 없으면 정상입니다!** ✅

---

## 3단계: GitHub에 저장소 만들기 (3분)

### GitHub 웹사이트에서 저장소 생성

1. 인터넷 브라우저 열기 (Chrome, Edge 등)
2. [github.com](https://github.com) 접속
3. 오른쪽 위 **Sign in** 클릭 (이미 로그인되어 있으면 생략)
4. GitHub 계정으로 로그인

5. 오른쪽 위 **+** 아이콘 클릭 → **New repository** 클릭

6. 저장소 설정:
   - **Repository name**: `qr-code-generator` (또는 원하는 이름)
   - **Description**: `QR 코드 생성기` (선택사항)
   - **Public** 선택 (무료로 사용 가능)
   - **Add a README file** 체크 해제 (이미 있으므로)
   - **Add .gitignore** 체크 해제
   - **Choose a license** 체크 해제

7. **Create repository** 버튼 클릭

8. **중요!** 다음 페이지에서 보이는 저장소 주소를 복사해두기:
   ```
   https://github.com/your-username/qr-code-generator.git
   ```
   (your-username은 여러분의 GitHub 사용자 이름입니다)

---

## 4단계: 코드를 GitHub에 업로드하기 (2분)

### 명령 프롬프트에서 실행

아직 명령 프롬프트가 열려있고 프로젝트 폴더에 있다면 그대로 사용,
닫았다면 2단계를 다시 따라해서 프로젝트 폴더로 이동하기.

다음 명령어들을 **하나씩** 입력:

```bash
git branch -M main
```

```bash
git remote add origin https://github.com/your-username/qr-code-generator.git
```
**⚠️ 주의**: `your-username`을 여러분의 실제 GitHub 사용자 이름으로 바꾸세요!

예시:
- GitHub 사용자 이름이 `johndoe`라면:
  ```
  git remote add origin https://github.com/johndoe/qr-code-generator.git
  ```

```bash
git push -u origin main
```

**마지막 명령어 실행 시:**
- GitHub 로그인 창이 뜰 수 있습니다
- **Username**에 GitHub 사용자 이름 입력
- **Password**에 GitHub 비밀번호 입력 (또는 Personal Access Token)
- 로그인하면 업로드가 시작됩니다

**성공 메시지 예시:**
```
Enumerating objects: XX, done.
Counting objects: 100% (XX/XX), done.
...
To https://github.com/your-username/qr-code-generator.git
 * [new branch]      main -> main
```

## 5단계: GitHub Pages 활성화하기 (3분)

### GitHub 웹사이트에서 설정

1. GitHub 저장소 페이지로 이동
   - [github.com](https://github.com) 접속
   - 여러분의 저장소 클릭 (예: `qr-code-generator`)

2. 저장소 페이지에서:
   - 상단 메뉴에서 **Settings** 클릭
   - (왼쪽 메뉴가 보이지 않으면 스크롤 다운)

3. 왼쪽 메뉴에서:
   - **Pages** 클릭 (맨 아래쪽에 있을 수 있음)

4. **Build and deployment** 섹션에서:
   - **Source** 드롭다운 클릭
   - **GitHub Actions** 선택

5. **Save** 버튼 클릭

6. 잠시 기다리기 (1-2분)

---

## 6단계: 배포 확인하기 (2분)

### 배포 상태 확인

1. 저장소 페이지에서 **Actions** 탭 클릭
2. 노란색 또는 파란색 원이 보이면 → 배포 진행 중 ⏳
3. 초록색 체크 표시가 보이면 → 배포 완료! ✅

### 배포된 사이트 접속하기

1. 저장소 페이지에서 **Settings** → **Pages** 다시 클릭
2. **Visit site** 버튼 클릭
   또는
   주소창에 직접 입력:
   ```
   https://your-username.github.io/qr-code-generator/
   ```
   (your-username과 qr-code-generator를 실제 값으로 변경)

3. 사이트가 열리면 성공! 🎉

## ❓ 자주 묻는 질문 (FAQ)

### Q1: "fatal: not a git repository" 에러가 나요
**A**: 프로젝트 폴더로 제대로 이동하지 않았습니다. 2단계를 다시 확인하세요.

### Q2: "remote origin already exists" 에러가 나요
**A**: 이미 연결되어 있습니다. 다음 명령어로 확인:
```bash
git remote -v
```
다시 연결하려면:
```bash
git remote remove origin
git remote add origin https://github.com/your-username/qr-code-generator.git
```

### Q3: "Authentication failed" 에러가 나요
**A**: GitHub 비밀번호 대신 Personal Access Token을 사용해야 할 수 있습니다.
1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Generate new token 클릭
3. 권한 선택 (repo 체크)
4. 생성된 토큰을 복사해서 비밀번호 대신 사용

### Q4: 배포가 실패했어요
**A**: Actions 탭에서 빨간 X 표시를 클릭해서 에러 메시지를 확인하세요.
- 대부분의 경우 코드에 문제가 있거나
- 빌드 설정이 잘못되었을 수 있습니다

### Q5: 사이트가 404 에러가 나요
**A**: 
1. 배포가 완료될 때까지 기다리기 (5-10분)
2. URL이 정확한지 확인 (대소문자 구분)
3. Settings → Pages에서 Source가 "GitHub Actions"로 설정되어 있는지 확인

### Q6: 저장소 이름을 바꾸고 싶어요
**A**: 
1. Settings → General → Repository name 변경
2. `vite.config.js` 파일의 base 경로도 변경 필요 (고급)
